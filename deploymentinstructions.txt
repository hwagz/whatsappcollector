WhatsApp Group Message Collector — Deployment Instructions
==========================================================

Target: Ubuntu/Debian Linux server (VPS or dedicated)


1. PREREQUISITES
----------------

Install Node.js 18 or later (via NodeSource):

    curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
    sudo apt-get install -y nodejs

Verify:

    node --version   # must be v18+
    npm --version

Install C++ build tools (required by better-sqlite3):

    sudo apt-get install -y python3 make g++

Install Chrome/Chromium system libraries (required by puppeteer's bundled Chromium):

    sudo apt-get install -y \
      libgbm1 libxss1 libasound2 libatk1.0-0 libatk-bridge2.0-0 \
      libcairo2 libcups2 libdbus-1-3 libexpat1 libfontconfig1 \
      libgdk-pixbuf2.0-0 libglib2.0-0 libgtk-3-0 libnspr4 \
      libnss3 libpango-1.0-0 libx11-6 libx11-xcb1 libxcb1 \
      libxcomposite1 libxcursor1 libxdamage1 libxext6 libxfixes3 \
      libxi6 libxrandr2 libxrender1 libxtst6 fonts-liberation


2. TRANSFER THE PROJECT
-----------------------

Option A — git (recommended):

    git clone <your-repo-url> /opt/whatsappcollector

Option B — copy files manually:

    scp -r /path/to/whatsappcollector user@yourserver:/opt/whatsappcollector

Do not copy node_modules/, .wwebjs_auth/, or messages.db.
The .gitignore already excludes these.


3. INSTALL DEPENDENCIES
-----------------------

    cd /opt/whatsappcollector
    npm install

This will also download Chromium (~150MB). Allow a few minutes.

If npm install fails with a node-gyp error for better-sqlite3, ensure
the build tools from step 1 are installed and retry.


4. CHROMIUM SANDBOX (important for most VPS providers)
-------------------------------------------------------

Most VPS hosts (DigitalOcean, Linode, Hetzner, etc.) do not allow
Chromium's kernel sandbox. If the service fails to start with an error
like "Running as root without --no-sandbox is not supported", add the
--no-sandbox flag to index.js.

Edit the Client constructor in index.js:

    const client = new Client({
      authStrategy: new LocalAuth(),
      puppeteer: { args: ['--no-sandbox', '--disable-setuid-sandbox'] },
    });

This is safe to apply proactively — do it before the first run to
avoid troubleshooting later.


5. FIRST RUN: SCAN THE QR CODE
-------------------------------

The WhatsApp session must be authenticated once interactively.
SSH into your server and run the app directly:

    cd /opt/whatsappcollector
    node index.js

A QR code will appear in the terminal. Open WhatsApp on your phone:

    Settings > Linked Devices > Link a Device

Scan the QR code. Wait for the "WhatsApp client ready" message.
The session is now saved to .wwebjs_auth/ on disk.

Press Ctrl+C to stop the process. You will not need to scan again
unless the session expires or .wwebjs_auth/ is deleted.


6. RUN AS A SYSTEMD SERVICE
----------------------------

Create the service file:

    sudo nano /etc/systemd/system/whatsappcollector.service

Paste the following (replace YOUR_USER and the path if different):

    [Unit]
    Description=WhatsApp Group Message Collector
    After=network.target

    [Service]
    Type=simple
    User=YOUR_USER
    WorkingDirectory=/opt/whatsappcollector
    ExecStart=/usr/bin/node index.js
    Restart=on-failure
    RestartSec=10
    StandardOutput=journal
    StandardError=journal

    [Install]
    WantedBy=multi-user.target

Enable and start the service:

    sudo systemctl daemon-reload
    sudo systemctl enable whatsappcollector
    sudo systemctl start whatsappcollector


7. MANAGING THE SERVICE
-----------------------

Check status:
    sudo systemctl status whatsappcollector

View live logs:
    sudo journalctl -u whatsappcollector -f

View recent logs:
    sudo journalctl -u whatsappcollector -n 100

Stop:
    sudo systemctl stop whatsappcollector

Restart:
    sudo systemctl restart whatsappcollector

Disable autostart:
    sudo systemctl disable whatsappcollector


8. DATABASE AND BACKUP
-----------------------

The database is at:

    /opt/whatsappcollector/messages.db

To inspect it:

    sqlite3 /opt/whatsappcollector/messages.db \
      "SELECT group_name, sender_name, body, datetime(timestamp,'unixepoch') FROM messages ORDER BY timestamp DESC LIMIT 20;"

To back it up (safe to do while the service is running — SQLite handles
concurrent reads gracefully):

    cp /opt/whatsappcollector/messages.db /backup/messages-$(date +%Y%m%d).db

The WhatsApp session is stored in:

    /opt/whatsappcollector/.wwebjs_auth/

Back this up too if you want to avoid re-scanning the QR code after
a server migration.


9. IGNORING GROUPS
-------------------

To prevent certain groups from being collected, edit config.json:

    nano /opt/whatsappcollector/config.json

Add group names exactly as they appear in WhatsApp (matching is case-insensitive):

    {
      "ignoredGroups": [
        "Family Chat",
        "Work Announcements"
      ]
    }

Restart the service to apply changes:

    sudo systemctl restart whatsappcollector

Group names are visible in the service logs (journalctl) as messages arrive.


10. UPDATING THE SERVICE
------------------------

    cd /opt/whatsappcollector
    git pull
    npm install
    sudo systemctl restart whatsappcollector
