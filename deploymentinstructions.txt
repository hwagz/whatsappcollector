WhatsApp Group Message Collector — Deployment Instructions
==========================================================

Target: Ubuntu/Debian Linux server (VPS or dedicated)


1. PREREQUISITES
----------------

Install Node.js 18 or later (via NodeSource):

    curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
    sudo apt-get install -y nodejs

Verify:

    node --version   # must be v18+
    npm --version

Install C++ build tools (required by better-sqlite3):

    sudo apt-get install -y python3 make g++

Install Chrome/Chromium system libraries (required by puppeteer's bundled Chromium):

    sudo apt-get install -y \
      libgbm1 libxss1 libasound2 libatk1.0-0 libatk-bridge2.0-0 \
      libcairo2 libcups2 libdbus-1-3 libexpat1 libfontconfig1 \
      libgdk-pixbuf2.0-0 libglib2.0-0 libgtk-3-0 libnspr4 \
      libnss3 libpango-1.0-0 libx11-6 libx11-xcb1 libxcb1 \
      libxcomposite1 libxcursor1 libxdamage1 libxext6 libxfixes3 \
      libxi6 libxrandr2 libxrender1 libxtst6 fonts-liberation


2. TRANSFER THE PROJECT
-----------------------

Option A — git (recommended):

    git clone <your-repo-url> /opt/whatsappcollector

Option B — copy files manually:

    scp -r /path/to/whatsappcollector user@yourserver:/opt/whatsappcollector

Do not copy node_modules/, .wwebjs_auth/, or messages.db.
The .gitignore already excludes these (including config.json itself).

After cloning/copying, create your config from the example:

    cp /opt/whatsappcollector/config.json.example /opt/whatsappcollector/config.json


3. INSTALL DEPENDENCIES
-----------------------

    cd /opt/whatsappcollector
    npm install --build-from-source=better-sqlite3

The --build-from-source flag is required on most Linux servers. The
prebuilt better-sqlite3 binary requires a newer glibc than many distros
ship (Ubuntu 20.04, Debian 11, etc.), so compiling from source avoids
a glibc version mismatch error. This also downloads Chromium (~150MB).
Allow a few minutes.

After installing, download the Puppeteer-managed Chrome browser as the
service user (it stores it in that user's home directory cache):

    cd /opt/whatsappcollector && sudo -u whatsappcollector npx puppeteer browsers install chrome

If the build still fails, verify the build tools from step 1 are installed:

    python3 --version && make --version && g++ --version


4. CHROMIUM SANDBOX (important for most VPS providers)
-------------------------------------------------------

Most VPS hosts (DigitalOcean, Linode, Hetzner, etc.) do not allow
Chromium's kernel sandbox. If the service fails to start with an error
like "Running as root without --no-sandbox is not supported", add the
--no-sandbox flag to index.js.

Edit the Client constructor in index.js:

    const client = new Client({
      authStrategy: new LocalAuth(),
      puppeteer: { args: ['--no-sandbox', '--disable-setuid-sandbox'] },
    });

SECURITY NOTE: --no-sandbox disables Chromium's process isolation. This
means that if a malicious WhatsApp message were able to exploit a browser
bug, it would have direct access to the server process. Mitigate this by:
  - Running the service as a dedicated low-privilege user (not root)
  - Using the systemd hardening directives in step 6
  - Keeping Node.js and npm dependencies up to date


5. FIRST RUN: SCAN THE QR CODE
-------------------------------

The WhatsApp session must be authenticated once interactively.
SSH into your server and run the app directly:

    cd /opt/whatsappcollector
    node index.js

A QR code will appear in the terminal. Open WhatsApp on your phone:

    Settings > Linked Devices > Link a Device

Scan the QR code. Wait for the "WhatsApp client ready" message.
The session is now saved to .wwebjs_auth/ on disk.

Press Ctrl+C to stop the process. You will not need to scan again
unless the session expires or .wwebjs_auth/ is deleted.


6. RUN AS A SYSTEMD SERVICE
----------------------------

WARNING: WhatsApp has a limit of devices you can be logged in to

sudo -u whatsappcollector node index.js

Create a dedicated non-root user to run the service (do not use root):

    sudo useradd --system --no-create-home --shell /usr/sbin/nologin whatsappcollector
    sudo chown -R whatsappcollector:whatsappcollector /opt/whatsappcollector

Lock down sensitive files so only that user can read them:

    sudo chmod 600 /opt/whatsappcollector/config.json
    sudo chmod 700 /opt/whatsappcollector/.wwebjs_auth

Create the service file:

    sudo nano /etc/systemd/system/whatsappcollector.service

Paste the following:

    [Unit]
    Description=WhatsApp Group Message Collector
    After=network.target

    [Service]
    Type=simple
    User=whatsappcollector
    WorkingDirectory=/opt/whatsappcollector
    ExecStart=/usr/bin/node index.js
    Restart=on-failure
    RestartSec=10
    StandardOutput=journal
    StandardError=journal

    ; Hardening
    NoNewPrivileges=true
    PrivateTmp=true
    ProtectSystem=strict
    ReadWritePaths=/opt/whatsappcollector

    [Install]
    WantedBy=multi-user.target

Enable and start the service:

    sudo systemctl daemon-reload
    sudo systemctl enable whatsappcollector
    sudo systemctl start whatsappcollector
    sudo systemctl restart whatsappcollector


7. MANAGING THE SERVICE
-----------------------

Check status:
    sudo systemctl status whatsappcollector

View live logs:
    sudo journalctl -u whatsappcollector -f

View recent logs:
    sudo journalctl -u whatsappcollector -n 100

Stop:
    sudo systemctl stop whatsappcollector

Restart:
    sudo systemctl restart whatsappcollector

Disable autostart:
    sudo systemctl disable whatsappcollector


8. DATABASE AND BACKUP
-----------------------

The database is at:

    /opt/whatsappcollector/messages.db

To inspect it:

    sqlite3 /opt/whatsappcollector/messages.db \
      "SELECT group_name, sender_name, body, datetime(timestamp,'unixepoch') FROM messages ORDER BY timestamp DESC LIMIT 20;"

To back it up (safe to do while the service is running — SQLite handles
concurrent reads gracefully):

    cp /opt/whatsappcollector/messages.db /backup/messages-$(date +%Y%m%d).db

The WhatsApp session is stored in:

    /opt/whatsappcollector/.wwebjs_auth/

Back this up too if you want to avoid re-scanning the QR code after
a server migration.


9. IGNORING GROUPS
-------------------

To prevent certain groups from being collected, edit config.json:

    nano /opt/whatsappcollector/config.json

Add group names exactly as they appear in WhatsApp (matching is case-insensitive):

    {
      "ignoredGroups": [
        "Family Chat",
        "Work Announcements"
      ]
    }

Restart the service to apply changes:

    sudo systemctl restart whatsappcollector

Group names are visible in the service logs (journalctl) as messages arrive.


10. RECAP EMAIL SETUP
---------------------

The recap script (recap.js) emails a summary of unread group messages.
It is run manually or via cron — it is not part of the main service.

Step 1: Generate a Gmail App Password

  Gmail requires an App Password (not your account password) for SMTP access.
  Your Google account must have 2-Step Verification enabled.

  1. Go to: myaccount.google.com
  2. Security → 2-Step Verification → App passwords
  3. Select "Mail" and your device, then click Generate
  4. Copy the 16-character password shown

Step 2: Fill in config.json

    nano /opt/whatsappcollector/config.json

Set the smtp password and add at least one recipient:

    {
      "ignoredGroups": [],
      "recipientEmails": ["recipient@example.com"],
      "smtp": {
        "host": "smtp.gmail.com",
        "port": 465,
        "user": "silverarrowweb@gmail.com",
        "password": "xxxx xxxx xxxx xxxx"
      },
      "aiApiKey": ""
    }

Step 3: Test manually

    cd /opt/whatsappcollector
    node recap.js
    manual run:
    node /opt/whatsappcollector/recap.js


If there are unrecapped messages in the database, an email will be sent.
If the database is empty, the script exits silently (no email).

Step 4: Set up a cron job

    crontab -e

Add a line for your desired schedule. Examples:

    # Daily at 8:00 AM
    0 8 * * * cd /opt/whatsappcollector && /usr/bin/node recap.js >> /var/log/whatsappcollector-recap.log 2>&1

    # Twice daily at 8 AM and 6 PM
    0 8,18 * * * cd /opt/whatsappcollector && /usr/bin/node recap.js >> /var/log/whatsappcollector-recap.log 2>&1

    # Every Monday at 9 AM
    0 9 * * 1 cd /opt/whatsappcollector && /usr/bin/node recap.js >> /var/log/whatsappcollector-recap.log 2>&1

To view the recap log:

    tail -f /var/log/whatsappcollector-recap.log


11. UPDATING THE SERVICE
------------------------

    cd /opt/whatsappcollector
    git pull
    npm install
    sudo systemctl restart whatsappcollector


12. RUNNING MULTIPLE INSTANCES
-------------------------------

You can run a second (or third) instance on the same server to collect
messages from a different WhatsApp account. No code changes are needed —
each instance is fully isolated by its directory.

What is automatically separate per instance:
  - .wwebjs_auth/   (WhatsApp session for that account)
  - messages.db     (separate database)
  - config.json     (separate group filters, email settings, etc.)

Steps for each additional instance (example: instance 2):

1. Clone or copy the project to a new directory:

       git clone <your-repo-url> /opt/whatsappcollector2
       cd /opt/whatsappcollector2
       cp config.json.example config.json
       npm install

2. Create a new dedicated system user:

       sudo useradd --system --create-home --shell /usr/sbin/nologin whatsappcollector2
       sudo chown -R whatsappcollector2:whatsappcollector2 /opt/whatsappcollector2

3. Install Chrome for that user:

       cd /opt/whatsappcollector2 && sudo -u whatsappcollector2 npx puppeteer browsers install chrome

4. Lock down sensitive files:

       sudo chmod 600 /opt/whatsappcollector2/config.json

5. Create a new systemd service file:

       sudo nano /etc/systemd/system/whatsappcollector2.service

   Paste (same as section 6 but with updated paths and user):

       [Unit]
       Description=WhatsApp Group Message Collector 2
       After=network.target

       [Service]
       Type=simple
       User=whatsappcollector2
       WorkingDirectory=/opt/whatsappcollector2
       ExecStart=/usr/bin/node index.js
       Restart=on-failure
       RestartSec=10
       StandardOutput=journal
       StandardError=journal

       ; Hardening
       NoNewPrivileges=true
       PrivateTmp=true
       ProtectSystem=strict
       ReadWritePaths=/opt/whatsappcollector2

       [Install]
       WantedBy=multi-user.target

6. Scan the QR code for the second WhatsApp account (interactive first run):

       cd /opt/whatsappcollector2 && sudo -u whatsappcollector2 node index.js

7. Enable and start the service:

       sudo systemctl daemon-reload
       sudo systemctl enable whatsappcollector2
       sudo systemctl start whatsappcollector2

MEMORY NOTE: Each instance runs its own Chrome process (~300-400MB RAM).
Plan accordingly if running several instances on a small VPS.
